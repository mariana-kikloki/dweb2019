Abstract

Neste TP foi desenvolvida uma plataforma CRUD (create, read, update, delete) com o objetivo de gerir o arquivo sonoro disponibilizado

Em primeiro lugar, converteu-se arquivo sonoro de XML para JSON, providênciando um ID para cada música, por forma a possibilitar as operações de apagar e modificar.

De seguida, procedeu-se à construção do CRUD, usando o express para facilitar o handle dos pedidos.

Para desenhar o front-end, utilizaram-se templates Pug, que são renderizados no servidor e enviados para o cliente quando é feito um pedido.
